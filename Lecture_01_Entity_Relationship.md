# 초롱대학교 데이터베이스 특강

## 제 1강: 데이터베이스의 시작, 엔티티(Entity)와 관계(Relationship) 찾기

**가상의 초롱대학교 환경**
* **학생:** 123명
* **학부:** 3개
* **학과(과):** 6개 (학부당 개)
* **교수:** 24명    

---

### 1. 개체(Entity) 찾기
개체(Entity)란 우리가 관리해야 할 **'핵심 정보의 대상'**을 말합니다. 명사 형태로 떨어지는 중요한 개념들입니다. 주어진 초롱대학교의 조건을 분석해 보면 다음과 같은 핵심 개체들을 뽑아낼 수 있습니다.

1. **학부 (Faculty)** : 6개의 학부가 존재합니다. (예: 인문대학, 공과대학 등)
2. **학과 (Department)** : 각 학부에 속한 3개의 전공을 의미합니다. (예: 컴퓨터공학과, 국어국문학과 등)
3. **교수 (Professor)** : 각 학과에 소속되어 학생을 가르치는 3~6명의 교수님들입니다.
4. **학생 (Student)** : 초롱대학교를 다니는 6,000명의 교내 핵심 구성원입니다.

### 2. 속성(Attribute) 부여하기
개체를 찾았다면, 이 개체들이 구체적으로 어떤 특징을 가지는지 정의해야 합니다. 이를 **속성(Attribute)**이라고 합니다. 

* **학부:** 학부코드(고유값), 학부이름, 학부장명, 사무실위치
* **학과:** 학과코드(고유값), 학과이름, 소속학부코드
* **교수:** 교번(고유값), 이름, 이메일, 전화번호, 소속학과코드
* **학생:** 학번(고유값), 이름, 학년, 입학년도, 소속학과코드, 담당지도교수_교번

> **💡 핵심 포인트 (Primary Key설정):**  
> 데이터베이스에서는 각 대상을 완벽하게 구분할 수 있는 **'고유한 식별자(기본키, Primary Key)'**가 반드시 필요합니다. 동명이인인 학생 2명을 구분하기 위해 고유한 **'학번'**을 부여하는 것이 그 예입니다.

### 3. 관계(Relationship) 설정하기
개체들이 독립적으로 떠도는 것이 아니라, 현실 세계처럼 서로 어떻게 연결되어 있는지 선을 그어주는 작업입니다.

* **학부 - 학과 간의 관계:** 하나의 [학부]는 여러 개의 [학과]를 포함합니다. (1 : N 관계)
* **학과 - 교수 간의 관계:** 하나의 [학과]에는 여러 명의 [교수](3~6명)가 소속됩니다. (1 : N 관계)
* **학과 - 학생 간의 관계:** 하나의 [학과]에는 여러 명의 [학생]이 소속됩니다. (1 : N 관계)
* **교수 - 학생 간의 관계:** 하나의 지도 [교수]는 여러 명의 [학생]을 지도할 수 있습니다. (1 : N 관계)

---

### 📝 요약 및 과제
물리적인 코딩을 하기 전, **1) 관리해야 할 대상(개체)을 찾고 2) 그들의 특징(속성)을 정의하며 3) 서로 간의 연결고리(관계)**를 도출해 보았습니다. 이것을 시각화하여 그림으로 그린 것을 **ER 다이어그램(ERD)**이라고 부릅니다.
