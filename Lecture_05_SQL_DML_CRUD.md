# 초롱대학교 데이터베이스 특강

## 제 5강: 데이터 조작어 (DML) - CRUD 실습

4강 마지막 퀴즈의 정답입니다! 완성된 빈 테이블(건물)에 새로운 학생 데이터(입주자)를 추가할 때 사용하는 명령어는 바로 **`INSERT`** 입니다.

이번 강의에서는 데이터베이스의 핵심 사이클인 **CRUD** (Create, Read, Update, Delete)를 수행하는 4가지 대표적인 **DML (Data Manipulation Language)** 매직 주문을 배워보겠습니다.

---

### 1. 데이터 삽입 (C - Create / INSERT)

초롱대학교의 `Student` 테이블에 새로운 1학년 신입생 '이데이터'를 등록해보겠습니다.

**기본 문법:**
```sql
INSERT INTO 테이블명 (컬럼1, 컬럼2, ...)
VALUES (값1, 값2, ...);
```

**실습 예제 (학생 추가하기):**
```sql
INSERT INTO Student (student_id, student_name, grade, entrance_year, department_code, professor_id)
VALUES (20240055, '이데이터', 1, 2024, 'CS01', 'P1553');
```
* **주의:** 문자열(문자, 날짜 형식 등)을 넣을 때는 반드시 양쪽에 **홑따옴표(`' '`)**를 씌워주어야 합니다. 숫자는 그대로 넣어도 됩니다.

### 2. 데이터 조회 (R - Read / SELECT) 👍 (가장 많이 씀!)

데이터베이스에 있는 수많은 정보 중, 내가 필요한 정보만 쏙쏙 뽑아볼 때 사용합니다. 실무에서 90% 이상의 작업을 차지하는 매우 중요한 명령어입니다.

**기본 문법:**
```sql
SELECT 가져올_컬럼1, 가져올_컬럼2 
FROM 대상_테이블명
[WHERE 검색_조건];
```

**실습 예제 (데이터 찾기):**
1. **모든 학생의 모든 정보 가져오기:** (`*`는 모든 컬럼을 의미합니다.)
   ```sql
   SELECT * FROM Student;
   ```
2. **학생들의 '이름'과 '소속학과코드'만 콕 찍어보기:**
   ```sql
   SELECT student_name, department_code FROM Student;
   ```
3. **조건 걸기! (입학년도가 2024년인 신입생만 찾아라):**
   ```sql
   SELECT * 
   FROM Student 
   WHERE entrance_year = 2024;
   ```
4. **조건 걸기 2! (컴퓨터공학과 'CS01' 소속 '2학년'만 찾아라):**
   ```sql
   SELECT * 
   FROM Student 
   WHERE department_code = 'CS01' AND grade = 2;
   ```

### 3. 데이터 수정 (U - Update / UPDATE)

입력된 정보를 고쳐야 할 때 사용합니다. '이데이터' 학생이 2학년으로 진급했습니다!

**기본 문법:**
```sql
UPDATE 테이블명
SET 고칠_컬럼 = 새로운_값
[WHERE 타겟이_누구인지_조건];
```

**실습 예제 (학생 진급시키기):**
```sql
UPDATE Student
SET grade = 2
WHERE student_id = 20240055;
```

> **🚨 초특급 주의사항:**
> 만약 실수로 `WHERE` 조건을 빼먹고 `UPDATE Student SET grade = 2;` 라고 친다면? 
> **초롱대학교 수천 명의 학생이 모두 2학년으로 강제 진급**해버리는 대형 사고가 발생합니다! (항상 타겟을 명확히 할 것!)

### 4. 데이터 삭제 (D - Delete / DELETE)

자퇴하거나 졸업한 학생의 데이터를 지워야 할 때 사용합니다.

**기본 문법:**
```sql
DELETE FROM 테이블명
[WHERE 타겟이_누구인지_조건];
```

**실습 예제 (특정 학생 기록 삭제):**
```sql
DELETE FROM Student
WHERE student_id = 20240055;
```

> **🚨 초특급 주의사항 2:**
> 이것 역시 `WHERE` 조건 없이 `DELETE FROM Student;` 를 실행하면, 그 즉시 **수천 명의 초롱대학생 기록이 흔적도 없이 전부 학살(삭제)** 됩니다! 백업본이 없다면 야반도주를 해야 할 수도 있습니다... 절대 주의!!

### 📝 요약 및 과제

5강에서는 데이터베이스의 꽃이자 가장 많이 쓰이는 **CRUD 사이클** (SELECT, INSERT, UPDATE, DELETE)의 기본을 배웠습니다. 
항상 잊지 마세요. 수정과 삭제는 돌이킬 수 없는 재앙을 부를 수 있으니 **반드시 `WHERE` 조건절**을 달아야 한다는 것을요!

**🗣 생각해 볼 문제 (마지막 퀴즈!):** 
교수님께서 이런 요청을 하셨습니다.
*"학생들 명표를 좀 뽑아주게! 이름, 학년, 그리고 **학과코드말고 진짜 학과이름**이 같이 나오게 해서 내 책상에 올려둬."*

그런데 우리는 학생의 이름(student_name)은 `Student` 테이블에, 학과이름(department_name)은 `Department` 테이블에 따로 분리해(정규화) 두었습니다.

이 두 개의 쪼개진 테이블에서 필요한 정보만 합쳐서 한 번에 조회하려면 특별한 기술(명령어)을 써야 합니다. 이 **테이블 결합 기술의 이름**은 무엇일까요?
